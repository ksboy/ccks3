# lic2020-ee

## 任务简介
事件抽取可以分成两步
1. 触发词词提取/事件类型：通常用NER的方式识别触发词，同时也判断出事件类型
2. 事件元素抽取：提取与触发词对应的事件元素

一般来说，触发词不存在重叠问题，但是可能存在多个相同类型的触发词（事件）；同一事件中事件元素可能存在重叠问题。

## 比赛思路
由于此次比赛只关注事件类型，并且同一文本中相同类型的事件合并不会影响评测结果，因此对于任务1，并不需要提取触发词。
除了NER方式，还可以采用**多标签分类**的方法判断事件类型（采用sigmoid激活函数）。

实验结果表明，NER和多标签分类对于事件类型分类结果差不多.

|                      | Precision          | Recall             | F1                 |
| -------------------- | ------------------ | ------------------ | ------------------ |
| NER                  | 0.9418052256532067 | 0.9571514785757392 | 0.9494163424124513 |
| multi-label-classify | 0.9548229548229549 | 0.943874471937236  | 0.9493171471927163 |

事件元素抽取我们尝试了BIO和start-end两种标注方式，后者有助于解决重叠问题。




### trick
对于任务1，多标签分类结果若为空，则选择score最大的标签。
